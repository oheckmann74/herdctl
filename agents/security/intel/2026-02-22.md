### Security-Relevant Changes

#### ✅ POSITIVE: Shell Escaping Fix (a0e7ad8)

**Commit**: `fix(core): escape $ and backtick in Docker CLI runtime prompts (#120)`

**What changed**:
```typescript
// Before (vulnerable):
const escapedPrompt = prompt.replace(/\\/g, '\\\\').replace(/"/g, '\\"');

// After (secure):
const escapedPrompt = prompt
  .replace(/\\/g, "\\\\")
  .replace(/\$/g, "\\$")      // NEW: Escape $
  .replace(/`/g, "\\`")       // NEW: Escape backtick
  .replace(/"/g, '\\"');
```

**Impact**: Prevents shell variable expansion and command substitution in prompts. Previously `$1234` would become `234` due to `$1` expansion.

**Status**: ✅ Security improvement - closes potential prompt injection vector

#### 🚨 NEW FINDING #011: Unauthenticated Web Dashboard APIs

**Severity**: MEDIUM-HIGH
**First Seen**: 2026-02-22
**Location**: `packages/web/src/server/`

**Attack Surface Introduced**:

| Endpoint | Method | Functionality | Auth |
|----------|--------|---------------|------|
| `/api/agents` | GET | List all agents with config | ❌ NONE |
| `/api/agents/:name` | GET | Get agent details | ❌ NONE |
| `/api/chat/:agentName/sessions` | GET/POST/DELETE | Manage chat sessions | ❌ NONE |
| `/api/chat/:agentName/messages` | POST | Send messages to agent | ❌ NONE |
| `/api/jobs` | GET | List all jobs | ❌ NONE |
| `/api/jobs/:jobId` | GET | Get job details/output | ❌ NONE |
| `/api/fleet/status` | GET | Fleet composition | ❌ NONE |
| `/api/schedules` | GET | List schedules | ❌ NONE |
| `/api/schedules/:name` | POST | Trigger schedule | ❌ NONE |
| `/ws` | WebSocket | Real-time event stream | ❌ NONE |

**Evidence**:
```typescript
// packages/web/src/server/index.ts:131-136
server.get("/ws", { websocket: true }, (socket, _request) => {
  // No authentication check
  wsHandler.handleConnection(socket).catch((error) => {
    logger.warn(`Error handling WebSocket connection: ${(error as Error).message}`);
  });
});

// packages/web/src/server/routes/agents.ts:18-37
server.get("/api/agents", async (_request, reply) => {
  // No auth guard
  const agents = await fleetManager.getAgentInfo();
  return reply.send(agents);
});
```

**CORS Configuration** (insufficient protection):
```typescript
// Only localhost origins allowed
origin: [
  "http://localhost:3232",
  "http://localhost:5173",
  "http://127.0.0.1:3232",
  "http://127.0.0.1:5173",
  `http://${config.host}:${config.port}`,
]
```

**Why This Is A Problem**:
1. **Information Disclosure**: Agent configs (Docker params, prompts, tool access) fully exposed
2. **Unauthorized Operations**: Anyone on localhost can trigger jobs, cancel schedules, create chat sessions
3. **No Rate Limiting**: Vulnerable to DoS via job spamming or WebSocket flooding
4. **Bind Configuration Risk**: Server can bind to 0.0.0.0 (network-accessible) via config with no auth enforcement

**Data Exposed**:
- Agent system prompts
- Docker configurations (volumes, environment, network settings)
- Job execution history and output (may contain sensitive data)
- Schedule definitions (cron expressions, triggers)
- Chat conversation history
- Fleet composition and agent status

**Impact Assessment**:
- **Confidentiality**: HIGH - Sensitive config and execution data exposed
- **Integrity**: MEDIUM - Can trigger jobs/schedules (but can't modify configs)
- **Availability**: MEDIUM - No rate limiting allows DoS

**Risk Level**: **MEDIUM-HIGH**
- Current deployment likely localhost-only (low exposure)
- If misconfigured to bind to network interface, becomes HIGH-CRITICAL

**Recommendation**:
1. **URGENT**: Design authentication framework (token-based or session)
2. **HIGH**: Add authorization checks per agent/resource
3. **HIGH**: Add rate limiting to all endpoints
4. **HIGH**: Default server to localhost-only binding with warnings if changed
5. **MEDIUM**: Add audit logging for all API operations
6. **MEDIUM**: Consider encryption for chat session persistence

**Priority**: **P1 - Address before widespread deployment**
