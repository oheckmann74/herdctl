### 3. Pattern Search Not Fully Executed

HOT-SPOTS.md defines 6 grep patterns to run during each audit:
1. Path construction without safety
2. Shell execution
3. eval/dynamic code
4. Secrets in logs
5. Docker capabilities
6. OAuth credential handling

**Evidence of execution**: The audit found Finding #011 (web APIs) which suggests manual review, but didn't document running the mandatory grep patterns from HOT-SPOTS.md.

**Missing**: No evidence that patterns 1-6 were systematically executed and results documented.

### 4. Attack Ideation Depth

The audit identified Finding #011 (web auth) with good impact assessment, but **attack scenarios are generic**:
- "Information disclosure" - not specific to herdctl data exposure
- "Unauthorized job triggering" - doesn't explore what a malicious job could do
- "DoS" - generic, no specific attack vector described

**Better attack ideation would include**:
- What sensitive data could be extracted from agent configs? (prompts, docker configs, env vars)
- Could an attacker trigger a malicious job that exfiltrates .herdctl/ state?
- Could WebSocket flooding exhaust file descriptors?
- Could job output endpoint leak credentials from previous agent runs?

**Rating**: Adequate (finds the vulnerability) but not Creative (doesn't explore novel combinations).

### 5. Finding #011 Investigation Incomplete

Finding #011 (OAuth credentials) was identified but not deeply investigated:

**What was done**:
- Listed code locations
- Identified 4 security concerns (permissions, logging, multi-user, lifecycle)
- Recommended 4 actions

**What was NOT done**:
- Did not grep for actual logger calls to verify token leakage risk
- Did not check if fs.chmodSync is already present
- Did not trace credential flow to verify lifecycle
- Did not verify if error messages contain sensitive data

This is noted in Questions Q12-Q14 but should have been investigated immediately rather than deferred.

### 6. Scanner Results Analysis Shallow

Scanner found **3 high severity npm vulnerabilities** (up from 2 moderate on 2026-02-22), but the audit:
- Did not investigate which packages are affected
- Did not assess severity or exploitability
- Just noted "Standard dependency warnings"

**Finding #008 still marked "Manual Check Needed"** - this should have been checked manually during this audit.
