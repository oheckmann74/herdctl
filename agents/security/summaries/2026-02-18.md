## Changes Since Last Audit (2026-02-17)

### New Finding: #011
`sessions.ts:415` uses `shell: true` when spawning `claude --resume <session_id>`. The `session_id` is read from disk state files and only validated as non-empty (no character restrictions). If a state file is tampered with, shell metacharacters in `session_id` could lead to command injection.

**Severity:** MEDIUM-LOW (requires prior write access to operator-controlled state directory)
**Fix:** Remove `shell: true` from spawn call (trivial)
